## How the deep-linking already works

Both `red/april-2026/index.html` and `black/april-2026/index.html` already implement both features. Here's how to use them:

## URL structure

```
https://yourdomain.com/red/april-2026/?rm=919205268482#partner-ekaa
```

> [!IMPORTANT]
> The `?` (query string) **must come before** the `#` (hash). Browsers strip the query string if the hash comes first.

---

## How each part works

### `#partner-ekaa` — Opens the accordion

The script at the bottom of each page (lines 558–574 in red, 708–723 in black) listens for the hash on load:

```js
window.addEventListener('load', () => {
  const hash = window.location.hash;           // → "#partner-ekaa"
  if (hash) {
    const targetCard = document.querySelector(hash); // finds <article id="partner-ekaa">
    if (targetCard && targetCard.classList.contains('partner-card')) {
      targetCard.classList.add('visible');      // makes it appear (bypasses scroll reveal)
      const header = targetCard.querySelector('.partner-header');
      if (header) {
        setTimeout(() => header.click(), 400); // opens accordion after 400ms
      }
    }
  }
});
```

Every `<article>` already has a matching `id`:

### `?rm=919205268482` — Wires the WhatsApp button

The RM logic runs immediately on page load (lines 576–589 / 725–738):

```js
const params = new URLSearchParams(window.location.search);
const rm = params.get('rm');  // → "919205268482"

document.querySelectorAll('.book-btn').forEach(btn => {
  if (!rm) {
    btn.style.display = 'none';   // no RM = button hidden entirely
  } else {
    const msg = encodeURIComponent(btn.dataset.message);
    btn.href = `https://wa.me/${rm}?text=${msg}`;
    // → https://wa.me/919205268482?text=Hi%2C%20I%20am%20a%20RedBeryl...
  }
});
```

Each button's pre-filled message is set via `data-message` in the HTML. The WhatsApp URL opens to a chat with that specific RM number and the correct restaurant's pre-filled text.
